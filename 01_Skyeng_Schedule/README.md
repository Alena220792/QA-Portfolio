# Проект: Тестирование модуля «Личные события» в расписании онлайн-школы Skyeng

Комплексный проект по тестированию веб-интерфейса и API для функционала «Личные события» в расписании преподавателей Skyeng.

## 📌 Краткое описание продукта
*   **Продукт:** Онлайн-школа Skyeng (Панель преподавателя).
*   **Раздел:** Расписание -> Личные события.
*   **User Story:** Предоставить преподавателям возможность гибко добавлять, редактировать и удалять личные события в расписании в зависимости от их графика.
*   **Заказчик:** ОАНО “ДПО “СКАЕНГ”.

---

## 🛠 Выполненные виды тестирования
*   Тестирование требований (статическое)
*   Smoke-тестирование & Приемочное тестирование (UAT)
*   Функциональное и регрессионное тестирование
*   Тестирование API (интеграционное)

## 📑 Тестовая документация и артефакты

### 1. Тест-дизайн (Ручное тестирование)
Разработана и пройдена тестовая модель в TMS **Test IT** (Метрика: **Пройдено 37/40 тест-кейсов — 88%**).
*   **Smoke-тесты:** Проверка критического пути создания/удаления событий.
*   **Приемочные сценарии:** Валидация разделения событий по цветам, создание личного события «задним числом», проверка установления длительности.
*   🔗 **[Посмотреть чек-листы и тест-кейсы в Test IT](ССЫЛКА_НА_ВАШ_TEST_IT)** *(или на Google Диск)*

### 🚀 2. Тестирование API
*   Проведено интеграционное тестирование бэкенда для логики создания и изменения параметров личных событий.
*   🔗 **[Скачать Postman-коллекцию проекта (postman_collection.json)](ССЫЛКА_НА_ФАЙЛ_В_РЕПОЗИТОРИИ)**

## 📊 Заключение и аналитический отчет (QA Sign-off)

**Решение о выпуске (Release Decision):** 
Выпуск продукта в продакшн **возможен**. Найденные дефекты не блокируют основные бизнес-сценарии использования расписания, однако требуют исправления в ближайших минорных релизах, так как могут снижать UX (пользовательский опыт).

**Анализ влияния дефектов на пользователей:**
1.  *Проблема с обновлением имени:* Требует ручной перезагрузки страницы пользователем для отображения актуального названия.
2.  *Ограничение дат в календаре:* Не критично, так как существует обходной путь (Workaround) — пользователь может перемещаться стрелками календаря до нужной даты и создавать событие кликом по ячейке.
3.  *Проблема с нестандартным цветом:* Логика выбора стандартных цветов в интерфейсе работает стабильно, проблема воспроизводится только при попытке передать кастомный hex-код.

**Рекомендации:**
1.  Устранить обнаруженные баги в порядке их приоритета (High -> Medium) в следующем спринте.
2.  После фикса дефектов провести полную проверку исправлений (Bug-fix Verification) и выборочный регресс.
3.  Включить автотесты для проверки API создания личных событий для предотвращения регрессионных ошибок в будущем.
