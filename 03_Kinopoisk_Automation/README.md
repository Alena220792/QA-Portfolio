# Документация проекта

## Описание базового синтаксиса записи и форматирования
Кинопоиск — крупнейший российский интернет-сервис о кино, сочетающий в себе подробную энциклопедию фильмов, сериалов и персон, а также онлайн-кинотеатр.

Проект использует следующие технологии и синтаксис:

1. **Python**: Основной язык программирования для написания тестов.
2. **Selenium**: Библиотека для автоматизации взаимодействия с веб-браузером.
3. **Pytest**: Фреймворк для написания и запуска тестов.
4. **Allure**: Инструмент для генерации отчетов о выполнении тестов.

### Форматирование кода

- Код форматируется в соответствии с PEP 8 (стиль написания кода на Python).
- Используются docstrings для документирования методов и функций.
- Все шаги теста размечаются с помощью `@allure.step` или `with allure.step` для улучшения читаемости отчетов.
- Используются декораторы `@allure.feature`, `@allure.story`, `@allure.title`, `@allure.description`, `@allure.severity`  для максимальной прозрачности отчета.
- Page Object Model (POM): Логика взаимодействия с элементами UI и методы API вынесены в отдельные классы в директории `pages/`.
- Разделение данных: Все URL, токены и учетные данные вынесены в `config.py`.
- Документирование: Методы снабжены **Docstrings** и **Type Hinting** (аннотации типов данных).
- Оптимизация: UI-тесты используют авторизацию через инъекцию куки `sessionid2` для обхода капчи.
- Команды для запуска тестов по отдельности:
API: pytest -m api test/test_api.py
UI: pytest -m ui test/test_ui.py

## Инструкция по запуску тестов для формирования отчета Allure

1. Установите необходимые зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. Запустите тесты с генерацией отчета Allure:
   ```bash
   pytest --alluredir=./allure-results
   ```

   Эта команда запустит все тесты и сохранит результаты в директорию `./allure-results`.

## Инструкция по просмотру сформированного отчета Allure

1. После завершения тестов, чтобы просмотреть отчет, выполните команду:
   ```bash
   allure serve ./allure-results
   ```

   Эта команда запустит локальный сервер и откроет отчет в браузере.

2. В отчете Allure представлены два основных блока:
2.1 Отчет по UI-тестам (Кинопоиск)
   - **Название теста**: `Проверка ключевого функционала сайта`.
   - **Описание теста**: `Тест проверяет поиск, навигацию по разделам и корректность отображения данных на страницах`.
   - **Шаги теста**:
     - Инициализация драйвера с скрытием признаков автоматизации.
     - Авторизация через подкладывание куки сессии из Config.- - Поиск фильма "Зеленая миля" с предварительной очисткой поля.
     - Переход в разделы "Онлайн-кинотеатр", "Сериалы" и "Билеты" через JS-клики.
     - Проверка наличия социальных сетей (ВКонтакте) в футере сайта.
   - **Результат**: Успешное выполнение или ошибка с указанием ожидаемого и фактического результата.
2.2 Отчет по API-тестам (Kinopoisk.dev) 
   - **Название теста**: `Поиск, фильтрация и безопасность API`.
   - **Описание теста**: `Тестирование эндпоинтов /movie и /search с валидацией JSON-ответов`.
   - **Шаги теста**:
     - Формирование запроса с авторизационным заголовком X-API-KEY.
     - Поиск фильма по названию и проверка вхождения в массив docs.
     - Получение детальной информации по ID (кейс 435).
     - Фильтрация по рейтингу 9-10 с циклической проверкой каждого фильма в ответе.
     - Негативная проверка: запрос с некорректным токеном (ожидаемый статус 401).
   - **Результат**: Успешное выполнение или ошибка с указанием ожидаемого и фактического результата.
